🇪🇨 Gastronomía de Ecuador - Smart Contract (Base Mainnet)
Este es el decimotercer contrato inteligente de mi colección profesional "Sabores de América", y el tercero de la etapa de Sudamérica. Ecuador, situado en el centro del mundo, ofrece una diversidad asombrosa que une la Sierra, la Costa y las Galápagos. Este contrato introduce la variable técnica de Guarnición, capturando la esencia de los acompañamientos tradicionales que definen la experiencia culinaria ecuatoriana.

🛠️ Detalles Técnicos: El Eje del Sabor On-Chain

Este despliegue sigue las mejores prácticas de desarrollo en Solidity:

Hito de la Serie: Decimotercer bloque completado, consolidando la presencia del proyecto en el cono sur.

Metadata de Guarnición (string guarnicion): Variable técnica diseñada para documentar el acompañamiento (ej. Canguil, Chifles, Tostado, Mote), un elemento de identidad regional muy fuerte en Ecuador.

Seguridad Auditada: Lógica sin manejo de fondos para prevenir vulnerabilidades financieras y asegurar un registro histórico íntegro.

Compilación Profesional: Implementación en Solidity ^0.8.20 con licencia MIT para transparencia y uso de código abierto.

📖 Modo de Uso

Consultar el Centro del Mundo: En la pestaña Read Contract, usa consultarPlato (ID 1 para el Encebollado) para ver la guarnición registrada por defecto.

Registrar Tradición: En la pestaña Write Contract, usa registrarPlato para añadir platos como "Fritada" (Guarnición: Mote y Tostado) o "Ceviche de Camarón" (Guarnición: Chifles).

Votación Inmutable: Califica los sabores mediante el sistema de Likes y Dislikes directamente en la blockchain.
